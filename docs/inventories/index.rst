Представление документов к требованиям
======================================

.. toctree::
   :maxdepth: 2
   :hidden:

   методы до описи

В предыдущих разделах были описаны :ref:`характеристики документооборотов<rst-markup-do>` и понятие входящего документооборота. Одним из примеров входящего документооборота является сценарий :ref:`получения требования<rst-markup-FNS-demand>` и :ref:`ответ на него<rst-markup-FNS-inventory>`.

По законодательству на некоторые требования необходимо отвечать в электронном виде с приложением файлов и их описанием. Чтобы соответствовать этим требованиям, был реализован новый тип документооборота — inventory (ответ на требование). После отправки ответа на требования, документооборот нужно отслеживать, обрабатывать ответные документы. В данном разделе описаны методы API Контур.Экстерна, которые позволяют отслеживать документооборот ответа на требование.
