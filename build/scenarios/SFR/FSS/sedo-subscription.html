Подписки СЭДО СФР на получение входящих документооборотов

Для получения входящих документооборотов от СФР в рамках СЭДО необходимо выполнить подписку оператора на организацию по РНС (тип документооборота urn:docflow:fss-sedo-provider-subscription).

Данная подписка является отдельными документооборотом. Процесс смены статусов документооборотов см. в разделе спецификация.

Создание и отправка подписок

Алгоритм создания и отправки черновика подписок

  1. Создать черновик. Метод POST Create draft. В метаинформации черновика необходимо заполнить поле registration-number-fss — регистрационный номер, по которому производится подписка.

  2. Создать документ в черновике по json контракту. Метод POST BuildDocumentContent. В ответе метод возвращает информацию о документе.

  3. Проверить черновик POST Check.

  4. Подготовить черновик POST Prepare.

  5. Отправить черновик POST Send.

После отправки подписки, СФР пришлет результат.

Важно

Для тестирования отправки подписок есть специальный робот, который эмулирует работу приемного комплекса на тестовой площадке. Для этого нужно в п.1 при создании черновика указать в recipient значение fss-code: SEDO-TEST.

Получение результата подписки

Для подписки оператора положительный результат от СФР поступит в виде документа в отправленном документообороте.

Для получения результата подписки оператора вам нужно найти документооборот и ожидать появления в нем результата подписки:

  1. Найти документооборот методом GET Docflows с фильтром type=fss-sedo-provider-subscription.

  2. В полученной метаинформации документооборота в поле success-state будет отображаться статус подписки. Если подписка выполнена успешно, то статус будет successful.

На этом документооборот завершится и подписка будет считаться выполненной.
